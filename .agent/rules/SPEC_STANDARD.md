SPEC standard
- Folder: .specs/<slug>/{requirements.md,design.md,tasks.md}
- requirements.md must include goals, scope, non goals, constraints, i18n, error handling.
- design.md must include data model, flows, IPC or API contracts, edge cases.
- tasks.md must be phased, layered, auditable.
- Do not invent behavior. Do not implement code in SPECs.
- All SPEC text must be ASCII only and English only when required by project rules.
